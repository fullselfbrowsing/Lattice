# v1.2 Milestone Integration Audit — FSB Integration + Agent Capability

## Scope

Cross-phase wiring + end-to-end user flows for milestone v1.2. Existence of files was verified during per-phase audits (Phase 14 + 15 + 16 + 17 + 18 retro cherry-pick equivalence; Phase 19 + 20 + 21 + 22 real-runtime). This report verifies CONNECTIONS only — that public exports from one phase are actually consumed by downstream phases and that user flows traverse the milestone's phase graph end-to-end.

The milestone shipped two tracks. Track A (Phases 14-18) backfilled five FSB-integration extensions onto canonical Lattice as retro cherry-picks. Track B (Phases 19-22) opened the previously out-of-scope Delegation surface as a runtime-agnostic single-agent capability. The integration surface to audit is therefore: (a) Track A exports consumed by Track B (bands, checkpoint hook, step.transition, provider parity, SurvivabilityAdapter), (b) Track A exports surfaced through the public index for external consumers, (c) Track B internal composition (runtime <-> host <-> infra primitives <-> eval helper <-> showcase).

## Audit Summary

- **All 46 v1.2 REQ-IDs WIRED.** No PARTIAL rows. No type-surface-only deferrals. Each REQ-ID is exercised by at least one real-runtime test or by the agent-loop showcase end-to-end.
- **Carryforward from v1.1 audit RESOLVED.** `TRACE-EXT-01` (recovery / eviction-resume markers in `RunEventKind` paired with `SurvivabilityAdapter`) was the one Important row left open by the v1.1 integration audit. Closed in Phase 20 via three additive literals (`recovery.start`, `recovery.complete`, `recovery.failed`) emitted by `runAgent` when host.storage.load() returns a snapshot at run start. Empirically asserted by `host-integration.test.ts` (fresh start, resume, corrupt snapshot).
- **Documented v1.2 limitation carried to v1.3: 1.** Native tool_use (Anthropic Messages-API `tools[]`, OpenAI Chat-Completions `tools[]`, Gemini `function_declarations`) is deferred. Phase 19 ships **prompt-reencoded mode uniformly across all 7 providers** because admitting native tool_use requires an additive extension to the `ProviderAdapter` interface that would risk the Phase 17 INV-03 7-provider parity contract. Recorded as `V1.2-LIMITATION-1` below; does not affect any v1.2 REQ-ID (AGENT-02 explicitly accepts prompt-reencoded uniformity).
- **Test count: 589/589 lattice core + 144/144 lattice-cli = 733/733.** Strict `tsc --noEmit` clean across both workspaces.

### Documented v1.2 Limitations

**V1.2-LIMITATION-1 — Native tool_use deferred across all 7 providers; prompt-reencoded mode uniformly.**

During Plan 19-03 implementation it surfaced that native tool_use protocols — Anthropic Messages-API `tools[]`, OpenAI Chat-Completions `tools[]`, Gemini `function_declarations` — cannot ship without modifying the `ProviderAdapter` interface in a way that risks breaking the Phase 17 INV-03 7-provider parity contract. The agent loop therefore ships **prompt-reencoded mode uniformly across all 7 providers** (openai, openai-compat, anthropic, gemini, xai, openrouter, lm-studio): conversation + tool descriptions + JSON-envelope instructions are encoded into the unchanged `ProviderRunRequest.task` field, and tool-call envelopes are parsed from the response text. This is documented as a deferral in Plan 19-03 SUMMARY and in `AGENTS.md`; the CONTEXT.md Q3 "native where supported" decision is treated as forward-compat intent for a follow-on milestone where the ProviderAdapter interface can be additively extended.

This does NOT affect any v1.2 REQ-ID. AGENT-02 explicitly accepts "implements a `tool_use` protocol that works across the v1.2 provider adapters" — uniform prompt-reencoded mode satisfies the contract. The 84-case `describe.each(ALL_PROVIDERS)` integration test in `format-tools.test.ts` proves parity.

**Carry-forward to v1.3:** When admitting native tool_use, the ProviderAdapter interface must be additively extended (e.g. optional `nativeToolUseCapability?: "anthropic" | "openai" | "gemini" | null` field) so the parity contract is preserved as a literal-union widening rather than a breaking change.
