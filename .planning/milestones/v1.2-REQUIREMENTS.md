# Milestone v1.2 Requirements — FSB Integration + Agent Capability

**Milestone:** v1.2
**Goal:** Land the five FSB-integration extensions (already implemented on `local-fsb-integration` branch, originating from FSB's v0.10.0-attempt-2 milestone) as canonical Lattice surface, and open the previously out-of-scope **delegation** surface by shipping a runtime-agnostic agent capability that any host application can consume — not just browser extensions.
**Created:** 2026-05-31

REQ-ID format: `<CATEGORY>-<NN>`. Numbering restarts for v1.2; v1.0 + v1.1 requirements are archived in `.planning/milestones/v1.0-*` and `.planning/milestones/v1.1-*`.

The milestone has two tracks:
- **Track A — FSB Integration (retro).** Code already exists on `local-fsb-integration` (HEAD `e95067b`), 23 commits, 414 vitest PASS. Each retro phase backfills GSD artifacts and merges the matching commit group into the `v1.2` branch via `--no-ff`. Source narrative for each REQ-ID is `automation/.planning/LATTICE-PIN.md` + `automation/lattice/docs/fsb-integration-gaps.md` from FSB v0.10.0-attempt-2.
- **Track B — Agent Capability (forward).** Net-new design. REQ-IDs declared here as intent; final shape locked during `/gsd-discuss-phase` for Phase 19 onward. The track opens the **Delegation** surface that v1.x previously declared out of scope.

### Capability Receipt v1.1 Schema Extension (RECEIPT-EXT)

- [x] **RECEIPT-EXT-01**: `CapabilityReceiptBody.version` widens from the v1.1 literal `"lattice-receipt/v1"` to the literal union `"lattice-receipt/v1" | "lattice-receipt/v1.1"`. Verifier accepts both. Sourced from FSB Phase 2 (commit `5c48134`).
- [x] **RECEIPT-EXT-02**: `CapabilityReceiptBody` gains six optional top-level fields — `stepName`, `stepIndex`, `parentStepName`, `previousStepName`, `sessionId`, `timestamp` — for step-marker linked-list threading. JCS canonicalization round-trip preserved. Redaction policy unchanged (step-marker fields are stable identifiers, not user content). Sourced from FSB Phase 2 (commit `5c48134`).
- [x] **RECEIPT-EXT-03**: `createReceipt` includes a `hasStepMarker` heuristic that auto-bumps emitted receipts to `lattice-receipt/v1.1` when any step-marker field is populated; legacy receipts continue to emit as `v1`. Sourced from FSB Phase 2 (commit `5c48134`).

### Tripwire / Hook Band Pipeline (BAND)

- [x] **BAND-01**: New module `packages/lattice/src/contract/bands.ts` exports `createHookPipeline()` factory returning a pipeline with priority bands `SAFETY=0`, `OBSERVABILITY=1`, `EXTENSION=2`. Lower number runs first; within-band registration order preserved. Sourced from FSB Phase 2 (commit `ba6172c`).
- [x] **BAND-02**: Per-handler `RegisterOptions.matcher` (optional regex) filters which lifecycle events a handler observes. Sourced from FSB Phase 2 (commit `ba6172c`).
- [x] **BAND-03**: Per-handler `RegisterOptions.budgetMs` (default 100ms) enforces a race-with-log timeout per handler. Timeout emits a `HOOK_TIMEOUT` event via the optional `TracerLike` sink. No-abort `Promise.race` semantics (CPU-leak risk explicitly accepted). Sourced from FSB Phase 2 (commit `ba6172c`).
- [x] **BAND-04**: `pipeline.run()` wraps each handler's context in `structuredClone` + `Object.freeze` so handler mutations do not leak across handlers. Sourced from FSB Phase 2 (commit `ba6172c`).
- [x] **BAND-05**: `pipeline.freeze()` is irreversible. Subsequent `register()` throws `Error` with `name === "PIPELINE_FROZEN"`. Sourced from FSB Phase 2 (commit `ba6172c`).

### Hook Lifecycle Events (LIFECYCLE)

## Track B — Agent Capability (forward)

### Delegation Surface Policy Flip (DELEG)

- [ ] **DELEG-01**: Lattice's `AGENTS.md` policy stance flips from `multi-agent crews: Out of Scope` to `agent execution: First-class, runtime-agnostic`. Decision record + rationale captured in v1.2 milestone audit.

### Agent Runtime Entrypoint (AGENT) — TBD during `/gsd-discuss-phase`

- [ ] **AGENT-01**: New module under `packages/lattice/src/agent/` exports an agent runtime entrypoint (e.g. `runAgent(...)` or `ai.runAgent(...)`) accepting a task description, tool registry, and host-adapter triple (scheduler + transport + storage). No coupling to `chrome.*`, `importScripts`, or service-worker idioms.
- [ ] **AGENT-02**: Agent loop implements a `tool_use` protocol that works across the v1.2 provider adapters (Anthropic native, OpenAI/OpenAI-compat, Gemini, xAI, OpenRouter, LM Studio).
- [ ] **AGENT-03**: Each agent iteration emits exactly one `step.transition` event via the existing `HookPipeline` so observability composes with `createCheckpointHook` (Receipts per iteration when a signer is configured).
- [ ] **AGENT-04**: Agent loop integrates with the band pipeline so a `band: SAFETY` handler can deny / abort an iteration before provider invocation.

### Pluggable Host Adapter (HOST) — TBD during `/gsd-discuss-phase`

- [ ] **HOST-01**: `AgentHost` adapter interface with three pluggable seams — scheduler (how iterations are scheduled — synchronous loop / `setTimeout` chain / queue), transport (how the provider call is dispatched — direct fetch / cross-process bridge / RPC), storage (how transcript and action history persist — memory / Redis / `chrome.storage` / filesystem).
- [ ] **HOST-02**: Reference `createNoopAgentHost()` implementation suitable for Node tests.
- [ ] **HOST-03**: SurvivabilityAdapter composes with AgentHost: the host's storage seam emits eviction snapshots through the SurvivabilityAdapter; on resume, the agent loop re-enters at the recorded step index.

### Cost / Transcript / Progress / History (AGENT-INFRA) — TBD during `/gsd-discuss-phase`

- [ ] **AGENT-INFRA-01**: Cost tracker primitive accumulates per-iteration `Usage` and exposes a running total; respects the existing `contract.budget` invariant from v1.1.
- [ ] **AGENT-INFRA-02**: Transcript store primitive records the message log; supports filtered tail reads for context-window management.
- [ ] **AGENT-INFRA-03**: Goal-progress tracker primitive offers a stuck-detection contract: caller declares a goal, tracker reports `progressing | stalled | regressed`.
- [ ] **AGENT-INFRA-04**: Action-history dedup primitive supports the "consecutive identical tool call" detector and the `STUCK_REASONS` vocabulary.

### Permissions (PERM) — TBD during `/gsd-discuss-phase`

- [ ] **PERM-01**: `PermissionContext` primitive lets callers gate tool execution per-tool, per-iteration, or per-resource pattern. Integrates with the band pipeline (typically `band: SAFETY`).

### Showcase + Eval (SHOWCASE-AGENT) — TBD during `/gsd-discuss-phase`

- [ ] **SHOWCASE-AGENT-01**: New `examples/agent-loop` showcase exercising the agent runtime end-to-end against a fake provider + fake tool registry; produces signed per-step receipts; replayable via `lattice repro`.
- [ ] **SHOWCASE-AGENT-02**: `lattice eval` adds an `--agent` mode or equivalent that gates a baseline-relative regression on iterations-to-goal and total cost.

## Out of Scope for v1.2

- Multi-agent crews (parent-child loops, summary-return, cache-prefix sharing, rate-limit-group coordination). Track B opens the **single-agent** surface only.
- Browser/extension-specific glue (offscreen-document host, `chrome.storage.session` survival, importScripts bundling). Stays in FSB.
- Mainline npm publish of `@fullselfbrowsing/lattice@1.2.0`. Tag + GitHub release only; npm publish deferred until at least one external consumer asks.
- Streaming for new provider adapters (Anthropic / Gemini / xAI / OpenRouter / LM Studio) — single-shot Promise only.
- Multi-model routing / fallback array on OpenRouter adapter.
- LM Studio latency-tail diagnostics module.
- Multimodal request shaping on Anthropic / Gemini adapters.
- OpenTelemetry tracing exporter for `RunEventKind`.
