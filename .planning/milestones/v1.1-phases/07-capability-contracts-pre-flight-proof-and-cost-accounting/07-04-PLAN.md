<objective>
Wire the contract carrier through the runtime, classify no-route failures into `no-contract-match` when contract reject codes are present, and populate `usage` on every `RunSuccess` and `RunFailure` return path. Export the contract factory and types from the package root so external consumers can author contracts.

This is the integration plan. After it lands, an end-to-end test can: (a) declare a `contract({ budget: { maxCostUsd: 0 } })`, (b) call `ai.run({ ..., contract })`, and (c) observe a `RunFailure` with `error.kind === "no-contract-match"`, `error.noRouteReasons[0].code === "contract-budget-exceeded"`, and `usage === { promptTokens: 0, completionTokens: 0, costUsd: 0 }`.

Purpose: This plan is the last mile — it converts the type-level contracts (Plans 01-03) into observable runtime behavior, gets the package back to a clean `pnpm tsc --noEmit`, and exports the public surface.

Output: Modified `runtime/create-ai.ts`, expanded `index.ts` exports, integration tests that exercise all four contract reject codes end-to-end through `ai.run`.
</objective>
