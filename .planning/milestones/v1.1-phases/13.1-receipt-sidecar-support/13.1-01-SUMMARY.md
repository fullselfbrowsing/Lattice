## Decisions Made

- **`isSidecarLoadError` discriminates on `kind`**, not on `path`/`message` only. The version-mismatch variant additionally requires `received: string` and unsupported-output-shape additionally requires `outputKey: string` — both verified by Test 13. This prevents accidental positive matches against generic typed-error objects from other loaders.
- **Walker error adaptation choice.** Non-file-not-found sidecar errors do NOT extend the consumer's `WalkedEntry` union with a new variant — instead they are downgraded to the existing `kind: "malformed"` shape (the original sidecar kind name is preserved in the message, e.g. `"sidecar version-mismatch: ..."`). This keeps `isWalkedReceiptError` truly binary and avoids forcing Plans 02/03 to re-narrow.
- **`classifyOutputSpec` returns `null` for invalid shapes.** Initial implementation used a `{ invalid: true }` sentinel, but TypeScript's `in` operator throws when the inspected value is a primitive (which is exactly the `"text"` happy path). Switched to a `null` discriminator — same total cost, no runtime hazard. (See Deviations.)
- **No build-emission expected for the new io/ modules.** `tsdown` tree-shakes from `src/cli.ts`. The io/ modules will land in `dist/*.js` only after Plans 02/03 import them from a command handler. Plan 01's success criteria only requires `tsc --noEmit` to be clean, which it is.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `classifyOutputSpec` returned a sentinel object incompatible with the `"text"` happy path**
- **Found during:** Task 1 (GREEN phase, first vitest run after implementation)
- **Issue:** The classifier returned `{ invalid: true }` for non-matching shapes and the caller used `if ("invalid" in classified)` to branch. JavaScript's `in` operator throws `TypeError: Cannot use 'in' operator to search for 'invalid' in text` when applied to a primitive string — which is exactly the literal `"text"` output, the most common happy path.
- **Fix:** Switched the classifier to return `null` on invalid shapes; caller now uses `if (classified === null)`.
- **Files modified:** `packages/lattice-cli/src/io/sidecar-loader.ts`
- **Verification:** Test 1 (happy path with literal text) now passes alongside all other 12 cases.
- **Committed in:** `2a358a4` (Task 1 commit)

**2. [Rule 1 - Bug] Test fixture's `validContract` was missing the mandatory `kind: "capability-contract"` discriminator**
- **Found during:** Task 1 (typecheck phase after vitest GREEN)
- **Issue:** `CapabilityContract` from `lattice` requires `kind: "capability-contract"`; the test had `{ capabilityId, invariants: [] }` which is structurally wrong for `lattice`'s `CapabilityContract` interface. `tsc --noEmit` failed with TS2741 in three test locations.
- **Fix:** Updated `validContract` to `{ kind: "capability-contract", invariants: [] } as const`.
- **Files modified:** `packages/lattice-cli/test/sidecar-loader.test.ts`
- **Verification:** `pnpm tsc --noEmit` now exits 0; all 13 loader tests still pass.
- **Committed in:** `2a358a4` (Task 1 commit)
