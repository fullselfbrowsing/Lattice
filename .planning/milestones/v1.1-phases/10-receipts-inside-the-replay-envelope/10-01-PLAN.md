<objective>
Phase 10 closes the offline-replay-plus-attestation cycle by making a `CapabilityReceipt` sufficient to materialize a deterministic `ReplayEnvelope`. After Phase 9, every terminal run branch emits a signed receipt; this plan teaches the replay subsystem to consume one.

Two things ship together because they are inseparable per CONTEXT.md:

1. `ReplayEnvelope` gains optional `receipt?: ReceiptEnvelope` and `contract?: CapabilityContract` fields, imported type-only so verify-only consumers do not pay for the receipt builder module graph.
2. A new pure async function `materializeReplayEnvelope(receipt, options)` that verifies the receipt first, then resolves content-addressed artifact bodies via a caller-supplied loader, then returns a `ReplayEnvelope` the existing `replayOffline()` already knows how to execute.

Purpose: Phase 11 (`lattice repro` CLI) is a thin shell over this function. The CLI loads the receipt from disk, plugs in a filesystem-backed `artifactLoader`, and calls `materializeReplayEnvelope`. The CLI cannot ship without this primitive, and this primitive is the only Phase 10 work (RECEIPT-09 is the sole requirement).

Output: A new `materialize.ts` module + co-located tests, augmented `ReplayEnvelope` type, two new public exports, and a round-trip test that proves `createReceipt -> materializeReplayEnvelope -> replayOffline` produces a result whose `outputHash` matches the original receipt.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| receipt source -> materializer | A `ReceiptEnvelope` arriving from disk, network, or any caller-supplied source is untrusted bytes until `verifyReceipt` returns ok. |
| artifactLoader output -> materializer | Loader-returned `ArtifactInput` bodies are content-addressed; the materializer trusts the loader to return bytes whose SHA-256 matches the receipt's `inputHashes` entries. CLI (Phase 11) verifies the hash before handing the input to the materializer; this plan documents the requirement but does not re-hash inside the kernel. |
| materializer output -> replayOffline | The produced `ReplayEnvelope` is consumed by `replayOffline`, which is itself a pure function. No new trust boundary. |
