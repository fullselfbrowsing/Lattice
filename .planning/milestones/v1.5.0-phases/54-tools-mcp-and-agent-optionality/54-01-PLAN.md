# Plan 54-01: Standalone Tools/MCP Helpers and Typed Agent Finals

<objective>
Make MCP/tool helpers concretely standalone by adding artifact conversion and boundary enforcement, then make `runAgent` return validated typed final outputs when callers declare output contracts.
</objective>

<requirements>
- TOOL-01: Developer can import MCP/tool helpers as standalone utilities without enabling the Lattice agent loop.
- TOOL-02: Developer can validate tool inputs and returned tool calls independently of provider execution.
- TOOL-03: Developer can turn MCP resources, prompts, and tool results into artifacts that can be packed, replayed, and signed.
- AGNT-01: Developer can opt out of all agent and crew surfaces while still using every non-agent Lattice module.
- AGNT-02: Developer can request typed agent final outputs when they do use `runAgent`, closing the BuildConfig-style object-return gap.
</requirements>

<threat_model>
- T-54-01: Tools-only imports accidentally reach agent or crew code. Mitigation: add `tools` to `scripts/check-lattice-module-boundaries.mjs` forbidden-agent checks and assert no `runAgent`/crew names in the tools facade.
- T-54-02: MCP helpers become tied to a specific SDK release. Mitigation: accept structural MCP-shaped records and do not import the MCP SDK.
- T-54-03: Typed agent output support changes default agent behavior. Mitigation: keep the default contract `{ answer: "text" }` and preserve existing no-outputs tests.
- T-54-04: Agent output validation duplicates runtime semantics and drifts. Mitigation: expose a shared output-map validation helper from `outputs/validate.ts` and reuse it in `runAgent`.
- T-54-05: Provider-native structured output forcing breaks intermediate agent tool-use envelopes. Mitigation: do not set `nativeStructuredOutput` automatically in the agent loop during this phase.
</threat_model>

<must_haves>
- MCP resource, prompt, and tool-result artifact helpers are exported from `@full-self-browsing/lattice/tools`.
- Returned tool-call validation remains importable and tested from the tools facade without provider or agent runtime.
- `scripts/check-lattice-module-boundaries.mjs` checks `tools.ts` against `src/agent/**`.
- `runAgent` sends declared output names to the provider and returns validated `InferOutputMap<TOutputs>` for success.
- `runAgent` returns failure kind `"validation"` when a declared final output fails validation.
- Existing `runAgent` callers with no `outputs` still receive `{ answer: string }`.
- Docs include a tools-only MCP artifact example and clarify that tools is covered by boundary checks.
</must_haves>
