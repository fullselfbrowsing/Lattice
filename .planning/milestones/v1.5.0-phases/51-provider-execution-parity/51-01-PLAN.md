# Phase 51 Plan: Provider Execution Parity

## Scope

Implement provider-only native execution parity as an additive provider contract.

## Tasks

### 51-01: Contract and Neutral Schema Helpers

- Add provider-native request types and finish metadata to `providers/provider.ts`.
- Add `tools/schema.ts` with a neutral Standard Schema to JSON Schema helper.
- Re-export the helper from `tools.ts`.
- Update `agent/format-tools.ts` to import/re-export the neutral helper.

### 51-02: OpenAI-Compatible and xAI Parity

- Serialize `nativeTools`, `nativeToolChoice`, and `nativeStructuredOutput` in Chat Completions request bodies.
- Parse buffered `message.tool_calls`.
- Materialize structured JSON output to the requested output slot.
- Attach finish metadata to buffered and streaming responses.
- Preserve unknown-but-live xAI model IDs with conservative live negotiation metadata.

### 51-03: Anthropic and Gemini Parity

- Serialize native tool declarations and tool-choice hints.
- Materialize Anthropic structured output through a synthetic forced tool.
- Serialize Gemini function declarations, tool config, and response schema hints.
- Parse buffered native tool calls and structured objects.
- Attach finish metadata to streaming complete chunks.

### 51-04: Tests and Compatibility Verification

- Add provider parity tests for native requests and response parsing.
- Add streaming collection coverage for finish metadata.
- Run package tests, typecheck, type tests, and module boundary check.
- Update roadmap/state/summary artifacts when verification passes.

## Out of Scope

- Changing `ai.run()` to automatically use native structured output based on `outputContracts`.
- Replacing the agent loop's prompt-reencoded tool protocol.
- Adding runtime Zod dependency for richer schema conversion.
- Guaranteeing every OpenAI-compatible gateway honors every native field.
