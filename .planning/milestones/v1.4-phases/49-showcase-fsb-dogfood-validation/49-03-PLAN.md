# Phase 49-03 Plan: FSB Package-Candidate Dogfood

## Objective

Add and run a non-mutating FSB dogfood runner that validates the v1.4 package candidate from packed tarballs, including new exports, version stamping, and receipt compatibility.

## Tasks

1. Add `scripts/dogfood-fsb-candidate.mjs` with configurable `--fsb-dir`, `--keep-temp`, and focused-test controls.
2. Pack the current runtime package and install it into an isolated temp project or temp FSB copy.
3. Validate candidate package behavior:
   - v1.4 public exports are present.
   - `latticeVersion` equals the packed manifest version and is not `0.0.0`.
   - receipt signing and verification still work from the packed candidate.
4. If an FSB checkout is present, run targeted FSB Lattice smoke tests from a temp copy without modifying the original checkout.
5. Print a stable summary suitable for the Phase 49 evidence matrix.

## Acceptance Criteria

- Original FSB checkout remains untouched.
- Runner fails clearly when the FSB checkout path is missing.
- Runner validates package-candidate install semantics rather than workspace symlinks.
- Output names the tarball version and each required dogfood assertion.
