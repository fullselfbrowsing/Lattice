# Phase 49-01 Plan: Offline v1.4 Validation Showcase

## Objective

Add an offline v1.4 showcase that exercises streaming, gateway metadata, OpenTelemetry export, and failure-mode behavior using fake providers and built package output.

## Tasks

1. Create a new example under `examples/v14-validation/`.
2. Implement deterministic scenarios:
   - Streaming collection through `policy.stream` or `collectStream`.
   - Gateway provider behavior with fake fetch-backed LiteLLM/OpenRouter-compatible surface.
   - OTel sink mapping with an in-memory tracer/span double.
   - Failure path with stream/provider failure and observable run events.
3. Add a focused test or script gate that spawns the example after the runtime build.
4. Add a root convenience script if useful for repeatable validation.

## Acceptance Criteria

- Example imports from built `packages/lattice/dist/index.js`.
- Example runs without network access, credentials, or local provider servers.
- Stdout includes parseable lines for all required scenarios.
- Failure-mode scenario verifies expected error/event behavior instead of merely catching any thrown error.

## Verification

- `pnpm --filter @full-self-browsing/lattice build`
- `node examples/v14-validation/index.mjs`
- Focused Vitest gate if added.
