# Plan 20-02: runAgent host integration + survivability resume

**Status:** Not started
**Wave:** 2 (depends on 20-01)
**REQ-IDs:** HOST-03

## Goal

Refactor `runAgent` to dispatch through host seams (with the noop default falling back to Phase 19 behavior). Wire `SurvivabilityAdapter` into the resume path: on run start, attempt to load a snapshot; on success, restore state and re-enter at the recorded step; on failure, emit `recovery.failed` and start fresh.

## Tasks

### Task 1: runAgent refactor

`packages/lattice/src/agent/runtime.ts`:
- Pull host from `intent.host ?? createNoopAgentHost()`.
- Pull `SurvivabilityAdapter` from `intent.survivabilityAdapter ?? createNoopSurvivabilityAdapter<AgentSnapshot>()`.
- Resume path (before first iteration):
  ```ts
  const snapshot = await host.storage?.load() ?? null;
  if (snapshot !== null) {
    intent.tracer?.event?.("recovery.start", { snapshotVersion: snapshot.version, capturedAt: snapshot.capturedAt });
    try {
      const state = await survivabilityAdapter.deserialize(snapshot);
      // restore iterationIndex, conversation, cumulativeUsage, providerName
      intent.tracer?.event?.("recovery.complete", { iterationIndex, providerName });
    } catch (error) {
      intent.tracer?.event?.("recovery.failed", { reason: error.message });
      await host.storage?.clear();
      // fall through to fresh start
    }
  }
  ```
- Transport dispatch: replace `provider.execute(req)` with `host.transport?.call(provider, req) ?? provider.execute(req)`.
- Scheduler yield: after each AFTER_AGENT_ITERATION, call `await host.scheduler?.scheduleNext(iterationIndex)`.
- Per-iteration save: after AFTER_AGENT_ITERATION emission (but before scheduleNext), serialize the current AgentSnapshot via `survivabilityAdapter.serialize(state)` and call `host.storage?.save(snapshot)`.
- On success (final-answer path): call `host.storage?.clear()` so the next run starts fresh.

### Task 2: Tests
