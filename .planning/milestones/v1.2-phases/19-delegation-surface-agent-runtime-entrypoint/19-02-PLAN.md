# Plan 19-02: Agent type definitions + budget + lifecycle event extensions

**Status:** Not started
**Wave:** 1 (independent — type/interface foundation)
**REQ-IDs:** (foundation for AGENT-01, AGENT-04)

## Goal

Land the type-level foundation for Phase 19: agent intent/result types, extend `BudgetInvariant` with two new fields, extend `HookLifecycleEvent` union with two new members, and define the deny-context shape used by `SAFETY`-band veto.

## Tasks

### Task 1: Agent types module — `packages/lattice/src/agent/types.ts`

Define and export:

- `AgentIntent` — `{ task: string; tools: readonly Tool[]; host?: AgentHost; contract?: CapabilityContract; policy?: RunPolicy; outputs?: OutputContract<any>; pipeline?: HookPipeline; signer?: ReceiptSigner; tracer?: TracerLike }`. `AgentHost` is forward-declared as `interface AgentHost { /* see Phase 20 */ }` — Phase 19 ships a default in-process scheduler/transport/storage internally; Phase 20 makes it pluggable.
- `AgentResult` — Discriminated union of `AgentSuccess` and `AgentFailure`. `AgentSuccess` carries `{ kind: "success"; output: T; usage: Usage; iterations: ReadonlyArray<IterationRecord>; receipt?: ReceiptEnvelope }`. `AgentFailure` reuses v1.1 `RunFailure` kinds plus new `agent-iteration-denied`, `agent-max-iterations`, `agent-wall-time-exceeded`.
- `IterationRecord` — `{ index: number; provider: string; promptTokens: number; completionTokens: number; costUsd: number | null; durationMs: number; toolCalls: ReadonlyArray<{ name: string; argsHash: string; resultHash: string }>; deniedReason?: string }`.
- `AgentDeniedError` — typed error for the `agent-iteration-denied` failure kind; carries `reason: string`. Inherits the same `terminal: true` semantics as `TripwireViolationError`.

### Task 2: Budget extension — `packages/lattice/src/contract/contract.ts`

Extend `BudgetInvariant` (current v1.1 shape) with two optional fields:
- `maxIterations?: number` — hard cap on agent iterations. Defaults to undefined; the runtime treats unset as "no limit" (caller responsibility to set).
- `maxWallTimeMs?: number` — hard cap on wall-clock time per `ai.runAgent` invocation.

Update the contract builder (`contract()` fluent factory) to accept these. Update `BudgetInvariant` JCS canonicalization order (alphabetical: `maxCostUsd`, `maxIterations`, `maxWallTimeMs`, `p95LatencyMs`).

### Task 3: Lifecycle event extension — `packages/lattice/src/contract/bands.ts`

Extend `HookLifecycleEvent` union additively:
- `BEFORE_AGENT_ITERATION` — emitted by `runAgent` before each iteration's provider call.
- `AFTER_AGENT_ITERATION` — emitted after each iteration completes (success, failure, or denial).

Both events carry `iterationIndex: number` and `intent: AgentIntent` on the context. `AFTER_AGENT_ITERATION` additionally carries `record: IterationRecord`.

### Task 4: Deny-context shape — `packages/lattice/src/contract/bands.ts`

Add a typed `HookDenyDirective = { reason: string }` shape. The pipeline gains a `pipeline.lastDenialReason(): string | null` accessor that reads the latest `context.deny` set during the most recent `run()` call. Each `run()` resets the prior denial state. (Composition with frozen-context: handlers write to `context.deny` on the structuredClone'd context; runtime reads from the clone before discarding.)
