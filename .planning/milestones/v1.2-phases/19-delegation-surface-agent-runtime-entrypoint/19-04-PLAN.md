# Plan 19-04: runAgent orchestrator

**Status:** Not started
**Wave:** 3 (depends on 19-02 types + 19-03 format-tools)
**REQ-IDs:** AGENT-01, AGENT-03, AGENT-04

## Goal

Ship `runAgent` as a method on the runtime returned by `createAI`. The orchestrator drives the iteration loop with sticky provider, deny detection, contract budget enforcement, auto-checkpoint registration when a signer is present, and per-iteration `step.transition` event emission.

## Tasks

### Task 1: `packages/lattice/src/agent/runtime.ts`

Export `runAgent(intent: AgentIntent, config: LatticeConfig): Promise<AgentResult>`.

Algorithm:

```
1. Construct or accept pipeline:
   pipeline = intent.pipeline ?? createHookPipeline(intent.tracer);
2. Auto-register checkpoint hook if signer present:
   if (intent.signer && pipeline.autoRegisterCheckpoint !== false) {
     pipeline.register(BAND.OBSERVABILITY, createCheckpointHook({
       signer: intent.signer, tracer: intent.tracer,
     }));
   }
3. Initialize loop state:
   - iterationIndex = 0
   - cumulativeUsage = { promptTokens: 0, completionTokens: 0, costUsd: 0 }
   - stickyProvider = null
   - startedAt = Date.now()
   - conversation = [{ role: "user", content: intent.task }]
   - iterations = []
4. Each iteration:
   a. Budget pre-check:
      - if (intent.contract?.budget?.maxIterations && iterationIndex >= max) →
          return AgentFailure { kind: "agent-max-iterations" }
      - if (intent.contract?.budget?.maxWallTimeMs && elapsed >= max) →
          return AgentFailure { kind: "agent-wall-time-exceeded" }
      - if (intent.contract?.budget?.maxCostUsd && cumulative >= max) →
          return AgentFailure { kind: "contract-budget-exceeded" }
   b. Emit BEFORE_AGENT_ITERATION:
      ctx = { iterationIndex, intent, conversation: structuredClone(conversation), deny: undefined };
      await pipeline.run("BEFORE_AGENT_ITERATION", ctx);
      if (pipeline.lastDenialReason() !== null) {
        record IterationRecord with deniedReason; emit AFTER_AGENT_ITERATION; return AgentFailure { kind: "agent-iteration-denied" }
      }
   c. Provider selection:
      - if stickyProvider, use it; else pick via deterministic router (same as ai.run).
   d. Format tools:
      const { requestFragment, parseToolUse } = formatToolsForProvider(provider.name, intent.tools);
   e. Call provider:
      Wrap in BEFORE_PROVIDER / AFTER_PROVIDER hook events (lifecycle defined in Phase 15).
      response = await provider.execute({ conversation, ...requestFragment, signal });
      Update cumulativeUsage.
   f. Parse tool calls:
      const toolCalls = parseToolUse(response);
      if (toolCalls === null) {
        // Final answer path
        Validate intent.outputs against response text (if outputs declared).
        Build AgentSuccess and exit loop.
      }
   g. Dispatch each tool call:
      For each ToolUseRequest:
        Wrap in BEFORE_TOOL / AFTER_TOOL hook events.
        tool = intent.tools.find(t => t.name === call.name)
        if (!tool) → push synthetic tool error result
        else: result = await runTool(tool, call.args)  // v1.0 primitive
        Wrap result via artifact.toolResult(...).
        Append to conversation.
   h. Sticky provider record (first success):
      if (!stickyProvider) stickyProvider = provider.
   i. Record iteration:
      iterations.push({ index: iterationIndex, provider: provider.name, ... });
   j. Emit AFTER_AGENT_ITERATION with the IterationRecord.
   k. iterationIndex++; loop.
5. Build receipt:
   If intent.signer present and final answer path:
     Mint outer-level receipt (sessionId = run id; embeds cumulative usage).
6. Return AgentResult.
```

### Task 3: Tests — `packages/lattice/src/agent/runtime.test.ts`

Real-runtime tests (no mocking Lattice primitives). Use `createFakeProvider({ capabilities })` from v1.0 and `defineTool` from v1.0.

Cases:
1. Single-iteration final answer (fake provider returns final answer immediately).
2. Two-iteration flow (fake provider returns tool_use → fake tool runs → fake provider returns final answer).
3. Sticky provider stays on first successful provider across iterations.
4. `SAFETY`-band handler sets `context.deny` → `AgentFailure { kind: "agent-iteration-denied" }` returned BEFORE provider invocation.
5. `OBSERVABILITY`-band `createCheckpointHook` auto-registered when signer present — per-iteration v1.1 receipt minted with `stepIndex` populated.
6. Auto-registration opt-out: `pipeline.autoRegisterCheckpoint = false` skips the auto-register; checkpoints not minted.
7. `maxIterations` budget enforced → `AgentFailure { kind: "agent-max-iterations" }`.
8. `maxWallTimeMs` budget enforced → `AgentFailure { kind: "agent-wall-time-exceeded" }`.
9. `maxCostUsd` budget enforced post-iteration → `AgentFailure { kind: "contract-budget-exceeded" }`.
10. `BEFORE_AGENT_ITERATION` / `AFTER_AGENT_ITERATION` events fire in order; payload carries `iterationIndex` + `record`.
11. `step.transition` event emitted once per iteration (via auto-registered checkpoint hook when signer present; otherwise emitted directly by the orchestrator).
12. Tool-call dispatch: tool result wrapped via `artifact.toolResult(...)`; receipt embeds artifact hash.
13. Final-answer schema validation: `intent.outputs` validated on the final assistant message; failure returns `AgentFailure { kind: "validation-failed" }`.
