    # Decisions traced to .planning/phases/25-pr-time-ci-workflow/25-CONTEXT.md:

      pull_request:

      push:

        name: ci
        runs-on: ubuntu-latest

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Type-check workspace

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| github.com -> runner | the workflow yaml controls what runs on a GitHub-hosted ubuntu-latest runner; misconfiguration is the TanStack May 2026 class of supply-chain risk |
| action vendor -> workflow | each `uses:` line invokes third-party JavaScript on the runner with the workflow's permissions; SHA pinning prevents tag-repointing attacks |
| PR author -> workflow | a malicious PR cannot escalate to publish capability because (a) no id-token write is granted, (b) no secrets are referenced, (c) pull_request_target is banned |
| workflow -> repository | the only permission granted is contents: read; the workflow cannot write to the repo, cannot push tags, cannot open issues |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-08 | Elevation of Privilege | every uses: line in ci.yml | mitigate | every uses: line is pinned by 40-character commit SHA (CI-02, D-12); a tag-repointing attack on actions/checkout, actions/setup-node, or pnpm/action-setup cannot reach this runner |
| T-25-09 | Spoofing | workflow trigger surface | mitigate | only pull_request and push are wired; pull_request_target is hard-banned by D-11 and enforced by scripts/check-workflow-safety.mjs; fork PRs cannot mint OIDC tokens because no job has id-token: write |
| T-25-10 | Information Disclosure | runner environment | accept | no secrets context referenced; no NODE_AUTH_TOKEN; runner has no production credential exposure |
| T-25-11 | Tampering | cache layer | mitigate | only the pnpm content-addressed store is cached (D-07); cache key hashes pnpm-lock.yaml only; no restore-keys fallback; no node_modules cache, no vitest cache, no build cache — cache poisoning surface is minimized per PITFALLS OIDC-1 cache-poisoning table |
| T-25-12 | Denial of Service | concurrency block | accept | PR-only cancellation means a flood of PR pushes cancels themselves cheaply; push-to-main is non-cancellable so rapid double-pushes to main do not orphan a green check (D-09 rationale) |
| T-25-13 | Repudiation | step-level naming | mitigate | step names follow the locked 11-step convention; each step's purpose is self-documenting in the Actions UI log; CONTEXT decision IDs cited in the workflow's header comment for audit trail |
| T-25-14 | Elevation of Privilege | future workflow drift | mitigate | scripts/check-workflow-safety.mjs runs in ci.yml itself so any future workflow that violates D-10/D-11 (e.g. a contributor adding pull_request_target or id-token: write outside a publish job in release.yml) fails ci before merge; this is the meta-defense that protects Phase 28's release.yml from regression |
</threat_model>
