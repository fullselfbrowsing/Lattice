<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Built-in regexes strip legitimate user-visible content | high | Keep patterns conservative; require no-op tests for normal text; avoid broad role/content deletion |
| Sanitizer context exposes secrets or raw provider payloads | high | Define context with only provider id, optional model id, and output name |
| Envelope extraction mutates or trusts parsed objects too much | medium | Parse JSON, read own properties by explicit path, never eval, no-op on ambiguity |
| Custom sanitizer failures disappear silently | medium | Let exceptions propagate through adapter execute; direct tests assert rejection behavior |
</threat_model>

    2. `export type SanitizerFn = (text: string, context: SanitizerContext) => string | Promise<string>;`
    3. `export type SanitizeOutputOption = SanitizerFn | readonly SanitizerFn[];`
    4. A helper for adapters:
       ```ts
       export async function applyOutputSanitizers(
         rawOutputs: Record<string, unknown>,
         sanitizeOutput: SanitizeOutputOption | undefined,
         context: Omit<SanitizerContext, "outputName">,
       ): Promise<Record<string, unknown>>
       ```
    5. If `sanitizeOutput` is absent, return `rawOutputs` unchanged.
    6. Normalize a single sanitizer to a one-item array.
    7. Apply sanitizer functions in array order to each string-valued output only.
    8. Preserve non-string output values exactly.
    9. Let sanitizer exceptions reject the returned promise; do not catch and mask them.
  </action>

  <read_first>

  </read_first>
  <action>
    Add these exports to `packages/lattice/src/sanitizers/sanitizers.ts`:

    1. `export function stripReasoningTags(): SanitizerFn`
       - Remove clearly delimited `<think>...</think>`, `<reasoning>...</reasoning>`, and `<scratchpad>...</scratchpad>` blocks.
       - Handle multiline blocks.
       - Trim only the final cleaned text.
       - Return original text when no known pattern appears.
    2. `export function stripChatTemplateArtifacts(): SanitizerFn`
       - Remove clearly exposed `<|im_start|>`, `<|im_end|>`, `[INST]`, `[/INST]`, `<<SYS>>`, and `<</SYS>>` markers.
       - Remove adjacent role labels only when attached to those markers.
       - Return original text when no known marker appears.
    3. `export function unwrapInternalEnvelope(schemaOrPath: string | InternalEnvelopeOptions | StandardSchemaV1): SanitizerFn`
       - Support `unwrapInternalEnvelope("summary")`.
       - Support `unwrapInternalEnvelope({ field: "summary" })`.
       - Support `unwrapInternalEnvelope({ path: "data.summary" })`.
       - Support a Standard Schema / Zod-like schema as a guard.
       - For schema-only input, extract only when the parsed object has exactly one own string field; otherwise no-op.
       - No-op on invalid JSON, missing path, non-object JSON, array JSON, and non-string extracted values.
    4. Define and export any minimal options type needed for `unwrapInternalEnvelope`, for example:
       ```ts
       export interface InternalEnvelopeOptions {
         readonly field?: string;
         readonly path?: string;
         readonly schema?: StandardSchemaV1;
       }
       ```
    5. Import Standard Schema as a type only if needed. Do not add a runtime schema dependency.
  </action>
  <verify>
    <automated>rg -n 'stripReasoningTags|stripChatTemplateArtifacts|unwrapInternalEnvelope|InternalEnvelopeOptions|StandardSchema' packages/lattice/src/sanitizers/sanitizers.ts</automated>

    - Missing or non-string extraction target returns the original text.
    - Built-ins do not reference Phase 34 failure mode enum values at runtime.
