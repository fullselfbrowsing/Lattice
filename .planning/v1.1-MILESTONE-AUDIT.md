## Tech Debt (carry-forward to v1.2)

1. **`lattice repro` against receipt-only envelope returns `execution_unavailable`** — receipt body is intentionally minimal (no task/outputs/policy). v1.2 will accept a sidecar JSON file with the original `RunIntent` inputs to enable end-to-end replay round-trips.
2. **`lattice eval` regression gate unreachable in v1.1** — same root cause; no fixtures reach `match` verdict because replay can't reconstruct the full intent. The mechanical pipeline (walker → verify → baseline diff → exit code) is fully wired and tested; v1.2's sidecar enables the gate to actually fire.
3. **Type-surface-only REQ-IDs** — CONTRACT-03 (qualityFloor), COST-02 (adapter normalization for all 3 families), EVAL-03 (N=3 judge median), EVAL-04 (judge cache) are covered by package-internal tests but not exercised by the work-inbox showcase. v1.2 showcase enrichment can demonstrate these end-to-end.
4. Vitest-compatible JSON/JUnit reporter for `lattice eval`.
5. Cross-platform CI matrix (Windows) + published-tarball smoke test.
6. Tripwires-as-eval-scorers wiring (report slot reserved).
7. Typed verify error union extensions (`EnvironmentDrift`, `RedactionDrift`, `Tampered`).
8. KMS adapter shapes for `ReceiptSigner`.
9. Lineage merkle root signed inside receipts.
10. Multi-tier cost/quality thresholds.
11. `lattice receipt diff` subcommand.
12. Real LLM judge implementations (caller-supplied; spec defined).

## Recommendation

**Accept tech debt and complete the milestone.** All 36 requirements satisfied; the two architectural limitations are documented, intentional, and have forward-compat hooks already in place for v1.2. No blockers exist.
