<task type="auto">
  <name>Task 2: Implement shared validator helper</name>
  <files>packages/lattice/src/tools/tool-call-validation.ts</files>
  <read_first>
    - packages/lattice/src/tools/tool-call-validation.ts

  <action>
    Add a shared helper with this public shape:

    ```ts
    export async function validateToolCallRequests(
      requests: readonly ToolUseRequest[],
      option: ValidateToolCallsOption | undefined,
    ): Promise<readonly ValidatedToolCall[] | undefined>
    ```

    Behavior:

    1. If `option` is undefined, return `undefined`.
    2. Build a `Map` of `option.tools` by `tool.name`.
    3. For each request, look up the tool by `request.name`.
    4. Missing tool creates `ToolCallValidationError` reason `"unknown_tool"` with `toolName = request.name`, `attemptedArgs = request.args`, empty `validationIssues`, and `requestId = request.id`.
    5. Existing tool validates `request.args` with `validateSchemaOutput(tool.name, tool.inputSchema, request.args)`.
    6. Schema failure creates reason `"invalid_args"` and carries normalized validation issues.
    7. Before or after schema validation, detect extra fields for Zod object schemas when possible. If extra fields are present and `allowExtraFields !== true`, create reason `"extra_fields"`.
    8. `"throw"` mode throws the first validation error in response order.
    9. `"drop"` mode omits invalid calls and returns valid calls.
    10. `"callback"` mode invokes `option.onValidationFailure(error)` once per invalid call and returns valid calls only.
    11. Validated calls preserve `id`, `name`, and validated/coerced args when schema validation returns a value.
  </action>
  <verify>
    <automated>rg -n 'validateToolCallRequests|validateSchemaOutput|unknown_tool|invalid_args|extra_fields|onFailure' packages/lattice/src/tools/tool-call-validation.ts</automated>
  </verify>
  <acceptance_criteria>
    - `validateToolCallRequests` returns `undefined` when option is undefined.
    - `unknown_tool`, `invalid_args`, and `extra_fields` appear in implementation.
    - Helper imports or otherwise uses `validateSchemaOutput`.
    - Throw mode throws the first error.
    - Drop and callback modes return only valid calls.
    - Callback mode invokes the callback once per invalid call.
  </acceptance_criteria>
  <done>Shared validator helper enforces tool registry and schema validation.</done>
</task>
