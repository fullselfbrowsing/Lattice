  <action>
    Fill the helper bodies with a strategy table or switch covering all five `RecommendedPromptStrategy` values.

    Required structured-output concepts by strategy:
    - `frontier`: concise contract phrasing; include `Return only content that satisfies the contract.`
    - `mid_tier`: explicit but not example-heavy; include `Treat the contract as instructions, not as prose to repeat.`
    - `reasoning`: include `Do not expose hidden reasoning, scratchpad text, or analysis in the final answer.`
    - `local`: include `Do not copy the contract, chat template, or wrapper text into the answer.`
    - `open_weight`: include all four exact strings:
      - `The contract below is an instruction, not text to output.`
      - `Do not answer with the schema, envelope, or any field name unless that field belongs in the final user-visible JSON.`
      - `Bad: {"summary":"Greeted the user."} when the user asked for a natural-language reply.`
      - `Good: Greeted the user.`

    Required tool-use concepts by strategy:
    - `frontier`: include `Use a tool only when the task requires an external action or lookup.`
    - `mid_tier`: include `The tool definitions are available actions, not answer text.`
    - `reasoning`: include `Keep tool selection separate from hidden reasoning.`
    - `local`: include `Do not invent tool names or arguments.`
    - `open_weight`: include all three exact strings:
      - `The tool list below is action metadata, not text to output.`
      - `Do not copy the tool descriptor into the final answer.`
      - `If no listed tool is needed, answer normally without fabricating a tool call.`

    The `switch` or table must be exhaustive over `RecommendedPromptStrategy`. If using a switch, include a `_exhaustive: never` default. If using `Record<RecommendedPromptStrategy, ...>`, the record type enforces exhaustiveness.
