# Phase 49-02 Plan: Tarball Native/Install-Script Validation

## Objective

Strengthen package tarball validation so the v1.4 core package proves optional integrations did not leak unwanted native dependencies or install lifecycle scripts.

## Tasks

1. Extend the existing tarball audit script or add a narrowly named sibling script.
2. Inspect packed `package/package.json` for:
   - install/preinstall/postinstall/prepare lifecycle scripts in published packages.
   - unwanted native/heavy optional dependencies in the core runtime package.
   - stale bare `lattice` references already covered by the existing gate.
3. Add or update a root script so the gate is easy to run in release validation.
4. Record the command and output in Phase 49 evidence.

## Acceptance Criteria

- Gate runs from repo root with zero external npm dependencies.
- Core runtime allowlist remains explicit and fails closed for native/heavy optional packages.
- CLI package remains allowed to depend on `@full-self-browsing/lattice`.
- Existing tarball leak behavior remains covered.

## Verification

- `node scripts/check-tarball-leak.mjs`
- Root package script if added.
