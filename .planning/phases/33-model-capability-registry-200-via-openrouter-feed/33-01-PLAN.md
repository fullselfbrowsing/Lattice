## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TypeScript compiler → runtime | Closed unions are erased at runtime; consumer code that violates them via `as` casts is the only injection vector. Mitigated by tsd type-tests + `noUncheckedIndexedAccess` + `satisfies` in the static arrays. |
| Public package surface → external consumers | Once published, every type in `src/index.ts` is a semver contract. Adding a member to a closed union (e.g., a new `KnownFailureMode`) is a typed breaking change. Documented in CONTEXT.md D-13 as the intentional gate for v1.4+. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01-01 | Tampering | KnownFailureMode union drift between Lattice versions | mitigate | tsd `assertExhaustive` test fails to compile if a mode is added without updating the switch — gates the breaking change at PR review time (Phase 33-RESEARCH §MUST-do Q11) |
| T-33-01-02 | Tampering | Consumer code casts an arbitrary string to `CapabilityAdapter` to bypass the closed union | accept | TypeScript intent is advisory at runtime; consumers wielding `as` cast accept the cost. Documented in PROJECT.md typing posture. |
| T-33-01-03 | Information disclosure | REQUIREMENTS.md edit accidentally exposes an unpublished CAPS-* decision | accept | REQ-ID specs are public-by-default in `.planning/`; no secrets in the prose; reviewer responsibility. |
| T-33-01-04 | Spoofing | Consumer claims to support a fake `CapabilityAdapter` value to confuse downstream lookup | mitigate | Closed union prevents `Map.get("not-a-real-adapter:foo")` at the type level; runtime `getCapabilityProfile` returns `undefined` for unknown keys (Plan 02 enforces). |
