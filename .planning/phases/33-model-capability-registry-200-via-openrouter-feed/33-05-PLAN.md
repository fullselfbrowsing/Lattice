<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub-hosted runner → OpenRouter | The cron job runs on a GitHub-hosted ubuntu-latest runner and fetches from a public endpoint. No secrets sent upstream; only the IP of the runner is observable to OpenRouter. |
| Generator output → committed branch | The runner pushes `chore/refresh-model-registry` to the repo using the auto-issued `GITHUB_TOKEN`. peter-evans/create-pull-request handles the push + PR-open. |
| peter-evans/create-pull-request action → repo | The action runs in the ubuntu-latest runner sandbox under `GITHUB_TOKEN`. SHA-pinned per CI-02 mitigates the action-supply-chain replacement risk (TanStack May 2026 OIDC compromise was an unpinned-version exploit). |
| `GITHUB_TOKEN` → repo permissions | Job-level `pull-requests: write` is the minimum needed; workflow-level `contents: read` locks the rest of the workflow down (so a future step extension cannot escalate without an explicit edit). |
