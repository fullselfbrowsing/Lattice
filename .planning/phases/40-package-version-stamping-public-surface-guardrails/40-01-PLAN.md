<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| T-40-01: manifest version changes but source version export remains stale | high | `stamp:version` runs before build/typecheck/test/test:types in source-consuming package scripts |
| T-40-02: CLI reports the runtime package version instead of the CLI package version | medium | CLI stamps `latticeCliVersion` from `packages/lattice-cli/package.json`; it does not import runtime `latticeVersion` |
| T-40-03: tests encode the next fixed literal, such as `1.3.0`, and go stale on the next release | medium | tests compare to package.json; tsd asserts string assignability, not a version literal |
| T-40-04: generated version module is nondeterministic and creates unnecessary diffs | low | generator writes a fixed header, one export line, and no timestamps |
</threat_model>

  <action>
    Create `scripts/stamp-package-version.mjs` as a zero-dependency Node 24 script using only `node:fs/promises`, `node:path`, and `node:url`. It must accept exactly these flags:
    `--package <path> --out <path> --export <identifier>`.

    Behavior:
    1. Read and parse the JSON at `--package`.
    2. Require `version` to be a non-empty string.
    3. Require `--export` to match `/^[A-Za-z_$][A-Za-z0-9_$]*$/`.
    4. Write `--out` with this exact shape, substituting the export name and JSON-stringified version:
       ```typescript
       // AUTO-GENERATED FILE - DO NOT EDIT.
       // Source: package.json version field.
       // Regenerate with this package's stamp:version script.
       export const latticeVersion = "1.3.0";
       ```
       The CLI package writes `latticeCliVersion` instead of `latticeVersion`.
    5. End the file with a single trailing newline.
    6. Do not write timestamps.
    7. On invalid args or invalid package JSON, print `[stamp-package-version] FAIL - <reason>` and exit 1.

    Update `packages/lattice/package.json` scripts to include:
    ```json
    "stamp:version": "node ../../scripts/stamp-package-version.mjs --package package.json --out src/version.ts --export latticeVersion",
    "build": "pnpm stamp:version && tsdown",
    "typecheck": "pnpm stamp:version && tsc -p tsconfig.json --noEmit",
    "test": "pnpm stamp:version && vitest run",
    "test:types": "pnpm stamp:version && vitest --typecheck --run && tsd"
    ```
    Keep `lint:packages` as `pnpm build && publint && attw --pack . --profile esm-only && node ./scripts/check-cli-deps.mjs`.

    Update `packages/lattice-cli/package.json` scripts to include:
    ```json
    "stamp:version": "node ../../scripts/stamp-package-version.mjs --package package.json --out src/version.ts --export latticeCliVersion",
    "build": "pnpm stamp:version && tsdown",
    "typecheck": "pnpm stamp:version && tsc -p tsconfig.json --noEmit",
    "test": "pnpm build && vitest run"
    ```
    Keep `lint:packages` as `pnpm build && publint && attw --pack . --profile esm-only`.

    Run both stamp commands once so `packages/lattice/src/version.ts` contains `export const latticeVersion = "1.3.0";` and `packages/lattice-cli/src/version.ts` contains `export const latticeCliVersion = "1.3.0";`.
  </action>
