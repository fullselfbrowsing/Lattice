# Plan 53-01: Standalone Core Preparation Record

<objective>
Add a non-executing `prepareCoreRun` helper that composes artifacts, optional storage, context packing, deterministic advisory routing, and execution-plan creation into one inspectable record for apps that already own model execution.
</objective>

<requirements>
- CORE-01: Context packing can be invoked directly over artifacts and optional session turns.
- CORE-02: Artifact constructors, refs, fingerprints, lineage, and packaging metadata are usable without provider execution.
- CORE-03: Deterministic routing and capability negotiation can be used as advisory APIs without invoking providers.
- CORE-04: Storage adapters are usable independently of `createAI()`.
- CORE-05: Standalone context/artifact/routing/storage modules emit inspectable plans or records suitable for receipts and debugging.
</requirements>

<threat_model>
- T-53-01: A "standalone" helper accidentally imports runtime, provider adapters, or agent code. Mitigation: place implementation under `src/core/standalone.ts`, use only existing pure kernels, and run `node scripts/check-lattice-module-boundaries.mjs`.
- T-53-02: The helper suggests a provider was executed. Mitigation: return `kind: "prepared-core-run"`, include advisory route/plan status only, and do not create provider responses, receipts, or attempts beyond existing planned/no-route plan shape.
- T-53-03: Hashes are fabricated for ref-only artifacts. Mitigation: expose available hashes only and keep per-artifact `inputHash` optional.
</threat_model>

<must_haves>
- `prepareCoreRun` exists and is exported from `@full-self-browsing/lattice/core`.
- `PreparedCoreRun.kind` is exactly `"prepared-core-run"`.
- The helper accepts optional `storage` and persists artifacts through `ArtifactStore.put` when supplied.
- The helper accepts optional `session` and passes it to `buildContextPack`.
- The helper accepts optional `catalog`; when omitted it returns a no-route plan with `catalogVersion: "standalone-empty"`.
- Tests prove route selection is advisory and no provider adapter execution function is called.
- Tests prove the helper returns artifact refs, context pack, route decision, execution plan, input hashes, and warnings.
- Type tests import `prepareCoreRun`, `PrepareCoreRunInput`, and `PreparedCoreRun` from `@full-self-browsing/lattice/core`.
- `docs/modular-entrypoints.md` includes a core-only example using `prepareCoreRun`.
</must_haves>

- default `catalog` to `{ version: "standalone-empty", models: [] }`,
- prepare each input artifact by either calling `input.storage.put(artifact)` or computing `fingerprintArtifactValue(artifact.value)`,
- build `routeDeterministically(catalog, { task, artifacts, outputs, policy, provider, model, contract })`,
- build `buildContextPack({ task, artifacts, route: route.selected, session, tokenBudget })` with optional fields omitted when undefined,
- create `createExecutionPlan({ task, artifacts: artifactRefs, outputs, route, context, metadata: { standaloneCore: true, ...metadata } })`,
- return `PreparedCoreRun` with `kind: "prepared-core-run"`, `version: 1`, `createdAt`, `task`, `outputNames`, `artifactRefs`, `artifacts`, `context`, `route`, `plan`, `inputHashes`, and `warnings`.
Export the helper and types from `packages/lattice/src/core.ts`.
</action>
