<!-- Plan 34-02 patterns to MIRROR (do NOT duplicate code; replicate the closure structure) -->

The Anthropic adapter implements:
- `cache: Map<modelId, {result, expiresAt}>` in closure
- `inflight: Map<modelId, Promise<T>>` in closure with `.finally` cleanup
- `async negotiate(modelId)`: cache-check -> inflight-check -> fetchAndNegotiate -> cache-write
- `async fetchAndNegotiate(modelId)`: retry-loop with backoff [0, 200, 1000]; auth-throw on 401/403; transient-fallback on others
- `function emitFallbackEvent(payload)`: optional RunEventSink

Plans 03 (this plan) and 04 (next plan) mirror this pattern. Each adapter's closure has its OWN cache + inflight. They differ ONLY in:
- /models URL path
- Auth header format
- Response body parsing (`mergeWithRegistry`)
