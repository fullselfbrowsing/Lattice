## Decisions Made
- The plan asked for the `contract()` import inside quality-floor.mjs to use a `canonicalize` import path. Since `canonicalize` is not on the lattice public surface, the scenario imports `packages/lattice/node_modules/canonicalize/lib/canonicalize.js` directly via a relative path. This is documented in the file header and keeps the public API surface unchanged (success criterion: "No new public API surface").
- The plan implied the single `ai.run` could route to the three real adapters. To keep ai.run deterministic without exercising real fetch/network paths through the router, the scenario wires a fake driver provider as the first provider on the `ai` instance and uses direct adapter.execute() introspection to surface the COST-02 normalizedUsage shape from each of the three adapter-family adapters. This matches the plan's stated intent ("THREE in-process introspection calls — directly call each adapter's execute(request) ...").

This matches the plan's stated rationale: "This intentionally bypasses the router — the goal here is to PROVE all three adapter families produce the normalized Usage shape (COST-02), not to route through them."
