<objective>
Harden noPublicUrl enforcement so the OpenAI-compatible adapter family (OpenAI, OpenRouter,
xAI, LM Studio, LiteLLM) cannot leak a public http(s) URL into the request body when
the packaging layer has blocked the `url` transport.

Purpose: The Gemini/Anthropic adapters honor `packaged.transport` per-artifact; the
OpenAI-compatible path does not — it serializes `url`/`value` raw regardless of transport.
This fix makes behavior consistent and adds a durable cross-adapter test guard.

Output:
- `multimodal.ts` — `isHttpUrl` promoted from private to `export function`
- `adapters.ts` — `createOpenAICompatibleRequestBody` gates `url` field on `transport === "url"`
  and gates public-URL string `value` on `transport === "url"`; imports `isHttpUrl` from
  `./multimodal.js` (no third copy)
- `adapters.test.ts` — new describe block: "noPublicUrl cross-adapter parity" with tests:
  (a) OpenAI-compat RED before fix / GREEN after,
  (b) Anthropic + Gemini parity lock,
  (c) positive: url IS emitted when transport is "url" (no over-blocking)
</objective>
