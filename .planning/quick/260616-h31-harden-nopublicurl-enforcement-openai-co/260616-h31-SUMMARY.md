# Quick Task 260616-h31 — Harden noPublicUrl enforcement (Summary)

## Goal

Close the `noPublicUrl` bypass in the OpenAI-compatible adapter found by the second local Codex review, and make enforcement consistent + regression-locked across all adapters.

## Audit result (pre-fix)

`noPublicUrl` is decided once in `packaging.ts` `chooseTransport` (blocks `url` transport and `file-id`-with-public-URL). `anthropic.ts` and `gemini.ts` already honor the per-artifact `packaged.transport` for every media branch, so they were compliant. The OpenAI-compatible `createOpenAICompatibleRequestBody` (`adapters.ts`) attached `transport` only as metadata and emitted `url`/`value` **raw** — so a public `http(s)` URL leaked into the request body under `noPublicUrl` for OpenAI / OpenRouter / xAI / LM Studio / LiteLLM. The gap had drifted because there was no cross-adapter test codifying the invariant.

## Changes

- **`adapters.ts`** — `createOpenAICompatibleRequestBody` now resolves the per-artifact transport once and gates URL emission on it: the `url` field is set only when `transport === "url"`; a string `value` is dropped when it is a public http(s) URL and `transport !== "url"`. Ordinary non-URL string content is unchanged. Gates purely on transport (same model as anthropic/gemini); no policy plumbing.
- **`multimodal.ts`** — promoted `isHttpUrl` to a shared `export` so `adapters.ts` reuses one copy (no third duplicate).
- **`adapters.test.ts`** — new `noPublicUrl cross-adapter parity` block (5 tests): OpenAI-compat blocked-`url` artifact, OpenAI-compat blocked value-borne URL, positive no-over-block (transport `url` → URL still emitted), plus Anthropic and Gemini parity locks. Artifacts built via `artifact.url`/`artifact.image` factories for type-safety.

## Verification

- The two OpenAI-compat parity tests are load-bearing: stashing the `adapters.ts` fix makes exactly those two fail; restoring → all pass. Anthropic/Gemini parity tests pass with or without the fix (confirms they were already compliant).
- Full CI mirror green: build, typecheck, test (1037), test:types, lint:packages, and all five `scripts/` audits. No version bump, no new changeset.
